What Is a Classification Tree?

🌳 The Big Idea

A decision tree partitions the feature space into rectangular regions using a series of binary splits. Each internal node asks a yes/no question about one feature. Observations flow down until they reach a leaf, which gives the predicted class.

✂️ How a Node is Split

At each node, the algorithm tries every feature and every possible threshold. It picks the (feature, threshold) pair that produces the greatest reduction in impurity — a measure of class mixing in each child node.

📐 Optimal Cutoff for Continuous Predictors

For a numeric variable X, sort the unique observed values. For each midpoint between consecutive values, compute the weighted impurity of the two resulting subsets. The cutoff that minimises weighted impurity wins.

For each threshold t: Left = {x : x ≤ t} Right = {x : x > t} Score(t) = |Left|/n · Imp(Left) + |Right|/n · Imp(Right) t* = argmin Score(t)

🔁 Recursive Partitioning

After finding the best split at the root, the same procedure repeats independently in each child node. This continues until a stopping criterion is met (max depth, min samples per leaf, or zero impurity).

Impurity Metrics

🎲 Gini Impurity

Probability that a randomly chosen observation is misclassified if labelled according to the class distribution in the node.

Gini(t) = 1 − Σ pᵢ² For binary: Gini = 1 − p² − (1−p)² = 2p(1−p) Range: [0, 0.5] (binary case) Max at p = 0.5 (50/50 split)

Used by default in scikit-learn's CART algorithm. Computationally fast — no logarithms.

📊 Entropy (Information Gain)

Average number of bits needed to encode the class label. Borrowed from information theory.

H(t) = −Σ pᵢ · log₂(pᵢ) For binary: H = −p·log₂p − (1−p)·log₂(1−p) Range: [0, log₂K] Max at uniform distribution (K = number of classes)

Information Gain = H(parent) − weighted H(children). Higher is better.

Which Metric to Choose?

🟢 Gini — Practical Choice

Slightly faster to compute. Tends to isolate the most frequent class into its own leaf. Works well for most binary classification tasks. Default in many libraries.

🔵 Entropy — Theoretical Depth

More sensitive to class distribution changes. Penalises impure nodes more heavily at intermediate impurity levels. Can produce slightly different tree shapes, especially in multiclass settings.
